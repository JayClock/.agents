请帮我用 TQA 编写用户故事验收条件。

## 字段说明

- 业务知识上下文：可留空；默认会读取 `.pi/user-story/glossary.md`、`domain-model.md`、`story-map.md`。如果本故事有特殊规则、旅程位置或边界，请补充。
- 用户故事：必填；推荐使用“作为 <角色>，我希望 <能力/功能>，从而 <业务价值>”。价值不要只复述功能。
- 相关用户故事 / 已知边界：可留空；填写上下游故事、依赖、明确不做范围、异常规则或风险。

## 示例

```text
业务知识上下文：
- 概念字典 / 关键规则：只有已支付订单才展示配送进度；已取消订单不显示配送进度。
- 领域模型 / 关键关系：客户拥有订单；订单关联商品；订单包含配送节点；物流记录记录已完成节点。
- 故事地图 / 当前故事所在用户旅程位置：订单中心 → 查看已支付订单 → 查看物流。

用户故事：
作为下单客户
我希望查看已支付订单的配送进度
从而安排收货或售后计划

相关用户故事 / 已知边界：
- 前置：客户已登录，订单已支付。
- 不做：商品推荐、发票生成、退款处理。
```
